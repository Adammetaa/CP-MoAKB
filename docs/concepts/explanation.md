# Explanation

An explanation is a deterministic structure built from explicit query matches,
validation issues, record evidence/status, or an unavailable request. It contains
facts, references, limitations, availability, and a stable summary. Rendering is
separate from construction.

Explanation improves inspectability; it does not guarantee factual correctness,
predictive accuracy, causality, diagnosis, or suitability for action. Missing
support remains visible rather than being filled by generative reasoning. RAS-006
and RAS-008 govern the model and projection.
