# Determinism

For equivalent explicit inputs and versions, CP-MoAKB preserves equivalent values,
ordering, projections, error classifications, and example output. It avoids clock,
random, host, environment, network, and hidden-state inputs in Runtime operations.

Determinism makes change review and traceability possible; it does not prove that
an input or scientific claim is correct. Build verification distinguishes byte-
stable wheels from normalized sdist comparison where archive mechanics differ.
