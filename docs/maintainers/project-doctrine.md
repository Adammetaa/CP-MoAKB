# Project Doctrine

Maintainer decisions apply these ordered preferences: knowledge before
intelligence; evidence before conclusion; explainability before automation;
official sources before unsupported claims; determinism before convenience;
traceability before hidden transformation; unknown before wrong; no hidden
intelligence; contracts before code; platform before product; versioned knowledge;
explicit authority; agricultural science before software convenience; ontology
before dataset; and long-term maintainability before short-term features.

These are review principles, not claims that explanation guarantees correctness or
that current outputs have empirical predictive accuracy. When principles conflict,
record the concrete tradeoff through existing governance rather than treating a
slogan as executable policy.
