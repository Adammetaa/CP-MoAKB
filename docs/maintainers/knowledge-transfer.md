# Knowledge Transfer

A maintainer handoff should identify current mission and non-goals, authoritative
versions, public manifest, active RAS/ADR set, Design Freeze, security reporting,
release state, verification commands, known limitations, and separately governed
future work. Link to repository evidence instead of relying on private memory.

New maintainers should run the quick start, one query/explanation, all verifiers,
and the full suite; then trace one feature from decision to contract, public
surface, tests, and implementation. Record ambiguity as an issue or decision
question rather than inventing context. The [traceability map](../governance/traceability-map.md)
is the starting exercise.
