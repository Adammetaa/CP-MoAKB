# Governance Boundaries

The Engineering Track governs code, APIs, deterministic behavior, dependencies,
packaging, security, tests, and developer documentation. The Knowledge Track
governs sources, terminology, authority, candidate records, scientific review,
and promotion. One track cannot silently approve the other.

Documentation inconsistency should be classified: if implementation and accepted
contract agree, correct prose; if implementation violates the contract, treat it
as a defect; if the contract is ambiguous, stop and seek a decision. Maintainers
must not manufacture contacts, reviewers, legal terms, source authority, or
scientific acceptance. See the [governance index](../governance/README.md).
