# Rejection Checklist

Reject or return a change when it introduces undocumented semantics, hidden
composition, non-determinism, unbounded input, private error leakage, forbidden
dependency direction, real agricultural claims without governance, or a manifest
change without compatibility analysis.

Also reject attempts to add diagnosis, recommendation, ranking, confidence,
scientific inference, AI/LLM behavior, persistence, production serving, secrets,
telemetry, or publication through an unrelated scope. Missing tests, unexplained
golden changes, disabled verification, broad allowlists, invented authority, and
claims unsupported by implementation are substantive blockers—not stylistic
preferences.
