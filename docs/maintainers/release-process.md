# Maintainer Release Process

Release preparation starts from a reviewed clean commit. Confirm all version
authorities, compatibility evidence, dependency review, security policy,
documentation, release notes, tests, verifiers, repeated builds, artifact contents,
and isolated installations. Use the [release handbook](../release/README.md) and
[readiness checklist](../release/release-readiness-checklist.md).

Preparation stops before destructive or external actions. Creating a tag, GitHub
Release, or package publication requires explicit final approval and appropriate
credentials outside these docs. No automated workflow is provided. Record what
was approved and verify the resulting public state separately.
