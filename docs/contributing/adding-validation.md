# Adding Validation

Runtime validation uses explicit rule metadata, profiles, context, stable issue
identifiers, severity, and deterministic ordering. A new rule must be mechanical,
testable, representation-neutral, and separately classified from scientific or
product acceptance.

Add focused positive, negative, ordering, profile, and architecture tests. Record
compatibility impact when a rule, identifier, severity, message, or ordering
changes. Do not modify frozen legacy semantic validation, infer truth from missing
data, promote candidates, or replace qualified review. RAS-002 is normative.
