# Philosophy

The project builds governed knowledge foundations before intelligent behavior.
It prefers traceable evidence to opaque conclusions, official sources to
unsupported claims, explicit limitations to guessed values, stable contracts to
convenient hidden behavior, and long-lived architecture to short-term feature
pressure.

“Explainability before accuracy” must not be read as permission to ignore
correctness. The intended meaning is that verifiability and traceability are
prerequisites for evaluating claims; the repository does not currently claim
empirical predictive accuracy at all. See the [project doctrine](../maintainers/project-doctrine.md).
