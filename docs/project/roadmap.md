# Roadmap

The immediate repository direction is stronger documentation, contributor
onboarding, deterministic examples, and continued compatibility/security
maintenance. Any later feature, knowledge-authoring tranche, deployment boundary,
or publication requires an explicit governed scope.

The broader [Strategic Roadmap 2.0](../ROADMAP_2.0.md) and conceptual architecture
describe possibilities, not commitments. Issue and sprint approval determine
actual work. Package `0.1.0` remains an alpha engineering baseline; no release or
tag is implied by this roadmap.

The [Knowledge Engineering Roadmap](../knowledge/roadmap/knowledge-engineering-roadmap.md)
separately sequences possible governance, authoring preparation, pilots, domain
population, and future implementation mapping. It is planning documentation only
and authorizes none of those activities.
