# Governance Maps

These maps connect repository evidence without inventing authority:

- [Decision map](decision-map.md)
- [Contract map](contract-map.md)
- [Traceability map](traceability-map.md)
- [Change classification](change-classification.md)

Project doctrine guides judgment; Design Freeze scopes what may change; ADRs
record accepted durable decisions; RAS documents define reusable Runtime
requirements; manifests enumerate governed surfaces; tests and scripts verify
implementation. Where no direct ADR-to-RAS relationship exists, the maps say so.
