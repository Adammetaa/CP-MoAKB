# Source Assessment Criteria

Each candidate source receives a narrative, claim-scoped assessment rather than a universal score.

| Dimension | Questions to record |
| --- | --- |
| Authority | Who publishes/owns the material, under what mandate, and for which claims? |
| Relevance | Which planned entities, fields, or relationships does it directly support? |
| Jurisdiction | Where does the claim apply, and is jurisdiction explicit or inferred? |
| Publication status | Is the item official, draft, archived, withdrawn, or unclear? |
| Currency | What publication/revision/effective dates exist, and what re-review trigger applies? |
| Traceability | Can a reviewer identify a stable document/page, version, location, and retrieval event? |
| Stability | Is the reference persistent; can content change without versioning? |
| Technical specificity | Does it define precise entities/relationships or only general navigation/guidance? |
| Review process | Is editorial, scientific, regulatory, or peer review documented? |
| Reuse constraints | Are copyright, license, quotation, image, and derived-data permissions clear? |
| Conflict risk | What terminology, taxonomy, version, scope, or policy conflicts are likely? |
| Extraction suitability | Are units clear, granularity consistent, tables/images necessary, and paraphrase feasible? |

## Assessment record minimum

Record publisher, title, canonical reference, source type, candidate claim scope, jurisdiction, visible version/date, retrieval date, current/archived/unknown state, expected extraction unit, metadata quality, rights status, limitations, required reviewers, assessment status, and reassessment triggers.

Missing metadata stays unknown. An assessment may approve a source for candidate extraction, require corroboration, restrict it to metadata, or reject/defer it. It never approves the extracted fact automatically.
