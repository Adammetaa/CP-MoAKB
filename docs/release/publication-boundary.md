# Publication Boundary

Tests, a green CI run, a local commit, and built artifacts are evidence—not
publication approval. Pushing source to the public repository requires explicit
approval. Creating a tag, GitHub Release, or uploading a package each requires a
separate explicit final decision and appropriate authority.

No current workflow requests secrets, OIDC, package-write permission, or release
permission. Documentation must not instruct maintainers to embed credentials or
imply that a package index already contains `cp-moakb`.
